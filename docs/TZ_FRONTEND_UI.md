# ТЗ: Frontend UI (фаза 0) — без бэкенда

Детальное описание интерфейса. Всё, что требует сервера, — **заглушки**.

---

## Принцип заглушек

| Действие | Поведение в UI-фазе |
|----------|---------------------|
| «Перейти к оплате» | Loader 1.5с → модалка «Оплата прошла» → заказ в localStorage |
| Отправка email | Текст «Отправлено на …» без реальной отправки |
| Webhook / банк | Не реализуется |
| SMS смена пароля | Кнопка → toast «Функция в разработке» |
| Админ-экспорт | Toast-заглушка |

**Реально работает:** навигация, формы, валидация, localStorage auth, загрузка файлов в браузер (preview + IndexedDB/localStorage метаданные).

## Дизайн-система

- **Цвета:** фон `#F8F6F3`, текст `#2C2C2C`, акцент `#2D4A3E` (тёмно-зелёный), вторичный `#6B7280`
- **Шрифты:** заголовки — `Playfair Display`, UI — `Inter`
- **Скругления:** карточки 12px, кнопка покупки — круг
- **Тени:** мягкие `shadow-md`, без резких границ

## PurchaseModal

### Поля

| Поле | Тип | Логика |
|------|-----|--------|
| Количество QR | number + ± | 1–50, влияет на итог |
| Email | email | required |
| Пакет | 3 radio-card | Стандарт 2990 / Премиум 5990 / Максимум 11990 |
| ФИО или организация | text | required |
| Сумма | readonly | `qty * price_per_package` |
| Адрес доставки | textarea | required |
| Телефон | tel mask +7 | required |

### Кнопка «Перейти к оплате»

1. Валидация Zod
2. Spinner 1.5с
3. `SuccessModal`: чек (мок), список QR (qrcode.react), инструкции
4. `localStorage.orders.push(order)`
5. Если залогинен — привязать к user, иначе предложить регистрацию

## Auth (модалка)

### Вход
- email, password
- Тест: `test@qr-memory.ru` / `1234`
- «Забыли пароль» → toast-заглушка

### Регистрация
- ФИО, email, телефон, пароль, повтор пароля
- Сохранение в `localStorage.users[]`
- Автовход после регистрации

## ЛК покупателя `/cabinet`

### Sidebar (desktop) / bottom tabs (mobile)

- Мои мемориалы
- Создать мемориал
- Мои заказы
- Настройки

### Редактор мемориала `/cabinet/memorial/:id`

**Блок 1 — Основное**
- ФИО усопшего
- Дата рождения / смерти (date picker)
- Папа / Мама (ФИО)
- Крылатая фраза (textarea, max 300)

**Блок 2 — Место захоронения**
- Карта (Leaflet, клик = маркер)
- Фото могилы (1 upload)

**Блок 3 — Фото**
- `UploadZone` multiple
- Прогресс: `23 / 40` (по пакету)
- Сетка превью, удаление
- Превышение лимита → блокировка + alert

**Блок 4 — Видео**
- Upload MP4/MOV
- Суммарные минуты: `12 / 20`
- Парсинг duration через `<video>`

**Блок 5 — Отзывы**
- Список + форма добавления
- Редактировать / удалить свои

**Блок 6 — Публикация**
- Кнопка «Сохранить» (localStorage)
- Кнопка «Предпросмотр» → `/m/:slug`

### Настройки
- Смена email (поле без отправки)
- Смена пароля
- Привязка QR-кода из заказа (select из orders)

## Админ `/admin`

Доступ: `user.role === 'admin'` или `admin@qr-memory.ru`

- **Dashboard:** карточки KPI + 2 графика Chart.js (мок)
- **Заказы:** таблица с фильтром по статусу
- **QR-коды:** slug, заказ, мемориал, сканы
- **Пользователи:** email, дата регистрации (без реального сброса пароля)
- **Отзывы:** одобрить / удалить
- **Аналитика:** выручка по месяцам (мок)

## UploadZone — спецификация

```
┌─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┐
  Перетащите фото сюда
  или [Выбрать файлы]
└─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┘
```

- Drag & drop highlight
- Список файлов с progress bar
- Валидация: тип, размер
- Фото: сжатие перед сохранением в IndexedDB
- Ошибки под каждым файлом

### Советы по оптимизации (встроить в UI)

Показать tooltip «Как ускорить загрузку»:
- Сжимаем фото автоматически
- Загружаем по 3 файла параллельно
- Видео лучше снимать в 1080p, не 4K

## Что НЕ делать в этой фазе

- PostgreSQL, FastAPI endpoints
- ЮKassa, webhooks
- Реальный SMTP
- Yandex S3 (только локальные blob URL)
- Микросервисы
- SMS-шлюз
